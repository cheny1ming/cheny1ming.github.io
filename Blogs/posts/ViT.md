## 1. 模型架构

了解了Transformer，相信大家对Bert模型都不陌生吧。ViT和Bert其实是有异曲同工之妙。Transformer
是 ViT 和 Bert 的共同基础，二者均围绕 Transformer
的编码器架构展开设计，核心差异在于对**输入数据的处理方式**——Bert
面向序列化的文本数据，ViT 则面向二维的图像数据。ViT 的设计本质是将 NLP
领域的 Transformer 适配到 CV 领域，其架构设计大量借鉴了 Bert
的思路，同时针对图像的二维特性做出了针对性改造。

### 1.1 Bert

![Bert](assets/images/ViT/Bert.png)

Bert 的全名是 Bidirectional Encoder Representations from Transformers，它的本质就是是基于 Transformer 编码器的预训练语言模型，其核心架构围绕**多头自注意力机制**和**前馈神经网络**构建，输入为经过分词、嵌入处理的一维文本序列。

上图（来源：菜鸟教程）主要展示了Bert模型的核心架构和预训练过程中的掩码语言建模（Masked Language Modeling, MLM）任务的具体流程。它基于的是Transformer编码器的架构，它天生就是双向的注意力，所以它不能和GPT那样做下一个词预测（Next Word Prediction），但它的优点就是可以轻松捕捉到文章之间的联系，掩码语言建模就类似于我们高中做的英语完形填空，就非常适配Bert模型。

接下来我们简单介绍一下Bert的模型架构：

- 嵌入层：由 Token Embedding（词嵌入）、Position Embedding（位置嵌入）和 Segment Embedding（句段嵌入）拼接而成，为每个文本 token 赋予语义、位置和句段信息；

- 编码器层：堆叠多层 Transformer 编码器，每层包含多头自注意力机制和前馈神经网络，通过自注意力捕捉文本中 token 之间的长距离依赖关系；

- 输出层：根据下游任务设计，如分类任务通过对 $[CLS]$ token
的输出做池化得到文本的整体表征。

Bert 的核心优势是通过自注意力机制摆脱了 RNN 类模型的序列依赖，能并行处理文本序列，同时有效捕捉长距离语义关联，这一优势也被 ViT 完整继承并适配到视觉任务中。

### 1.2 ViT

![ViT](assets/images/ViT/VIT.png)

ViT 直接沿用了 Transformer 编码器的核心结构，是不是跟Bert简直一模一样？仅针对图像的二维结构做了**序列化改造**和**输入层适配**，其整体架构可分为**输入处理层**、**Transformer编码器堆叠层**和**下游任务输出层**三部分：

- 输入处理层：将二维图像切分为固定大小的 patch（图像块），将每个 patch 展平为一维向量，完成图像的序列化，再通过嵌入层为其赋予语义和位置信息；

- 编码器层：与 Bert 一致，堆叠多层 Transformer 编码器，通过多头自注意力机制捕捉图像 patch 之间的空间关联和语义依赖；

- 输出层：与 Bert 的 $[CLS]$ token 类似，ViT 在输入序列中添加 $[CLS]$ token，最终通过对该 token的输出做线性变换得到图像的分类结果，针对检测、分割等任务则会做相应的输出层改造。

简单来说，ViT 的架构设计逻辑是**"把图像变成文本一样的一维序列，再用Transformer 编码器进行处理"**，这一极简的设计思路让 ViT在多个视觉任务中实现了超越 CNN 的性能。

## 2. 从图像变Token

Transformer的期望输入是一个二维矩阵 $（len, d_{model}）$，len是序列的长度，d_{model} 则是序列中每一个token的维度。对于Bert来说，很简单，只需要把一句话输入embedding层，自然能得到这个二维矩阵，可对于图片来说呢？图像是二维的像素矩阵，而Transformer编码器仅能处理一维序列数据，因此**将图像转化为一维的Token**是 ViT的核心预处理步骤，也是连接二维图像和 Transformer 的关键桥梁。

### 2.1 图像变patch

文字是一维的，可是图像是二维的呀，ViT是怎么做到把图像也变成 $（len, d_{model}）$ 并送入Encoder的呢。看好了，创新点来了，注意上图的这一图像预处理部分：

![Patch](assets/images/ViT/patch.png)

ViT的预处理把图像分成了patch！ViT把图片分成了几个小块，每一个小块称为patch。在原始论文中，输入图片的尺寸为：
$$224 \times 224 \times 3(H \times W \times C)$$

其中高（H）和宽（W）为 224 像素，通道数（C）为 3（RGB）。

每一个patch的尺寸设置为16，则每一个patch的图片大小为：$16 \times 16 \times 3$。一共就有 $ (224/16) \times (224/16) = 196 $ 个patch。

你肯定会问，那为什么要转化成patch呢，而且还得是特定大小的patch？别急，听我慢慢道来。

### 2.1 Patch 变 Token

接着我们再来看每一个patch的内部，patch的高和宽都是
16，通道数量为3（RGB），像素总数量就是 $16 \times 16 \times 3 =
768$。诶，每一个patch有768个像素，好熟悉的数字，那不是可以正好作为token的维度d\_model吗，所以每个patch就可以顺理成章的进入Transformer进行处理了。✌

让我再来回顾一遍详细的过程。Patch 变 Token 的过程本质是**图像的分块与展平**，整个过程无复杂的特征提取，仅为纯几何操作，具体步骤如下：

1. **图像标准化**：将输入图像统一缩放到固定尺寸，如 ViT-Base 采用的
224×224、256×256，确保后续分块的统一性；
2. **固定尺寸分块**：将标准化后的二维图像按照固定大小切分为互不重叠的正方形Patch，例如将 224×224 的图像切分为 16×16 的Patch，可得到(224/16)×(224/16)=196个 Patch；
3. **Patch 展平**：将每个二维的 Patch（如 16×16×3 的 RGB 图像Patch）按行或列展平为一维向量，16×16×3 的 Patch 展平后得到 768维的一维向量，最后的输入维度就是(196, 768)啦；
4. **Token 化**：展平后的每个一维 Patch 向量即为 ViT 的一个**ImageToken**，对应 NLP 中 Bert 的文本 Token，所有 Patch 对应的 Token按空间顺序排列，形成一维的 Token 序列，完成图像的序列化。

以 224×224×3 的 RGB 图像、16×16 的 Patch 大小为例，最终将得到 196 个 768维的 Image Token，形成长度为 196 的一维序列。

### 2.2 Patch 的必要性 
我第一次看到这里的时候，其实脑子里也有疑问：为什么要处理成patch再处理成token呢，我直接用图片的像素作为token一步到位岂不更好？这里呢，我说一下我的看法。

将图像切分为 Patch 并转化为 Token，并非简单的形式改造，而是适配Transformer 架构、兼顾计算效率和特征表达的必然选择，核心原因有三点：

- **适配 Transformer 的输入要求**：Transformer编码器的核心是自注意力机制，仅能处理一维序列数据，无法直接接收二维像素矩阵，Patch化的序列化是让 Transformer 处理图像的前提；
- **降低计算复杂度**：自注意力机制的计算复杂度与序列长度的平方成正比，若直接将图像的每个像素作为一个Token，224×224 的图像将产生 50176 个 Token，计算量会呈指数级飙升。而Patch 化将多个像素整合为一个 Token，大幅缩短序列长度，有效降低计算成本；
- **保留图像的局部空间特征**：图像的语义特征多体现在局部像素的关联中，如眼睛、嘴巴组成人脸，车轮、车身组成汽车。固定大小的Patch 能保留图像的局部空间信息，每个 Patch作为一个基本的视觉单元，其展平后的向量可作为局部特征的初步表征，为后续自注意力机制的全局特征融合奠定基础。

此外，Patch 的大小是 ViT 的重要超参数，Patch越小，序列长度越长，能保留的细节信息越多，但计算量越大；Patch越大，序列长度越短，计算效率越高，但可能丢失精细的局部特征，需根据任务需求权衡选择。

## 3. **Embedding层**

我们都知道，在文本Transformer中，比如Bert，GPT，都要添加一个嵌入层，为了保留文字本身的位置信息。可想而知，ViT也是需要的。

![Embedding](assets/images/ViT/embedding.png)
将图像转化为一维的 Image Token 序列后，这些 Token仅为像素的数值组合，不包含任何语义信息和空间位置信息。ViT通过**Embedding 层**为 Token 赋予语义和位置信息，这一步与 Bert的嵌入层设计思路一致，核心包含 Token Embedding 和 Position Embedding两部分，二者相加得到 ViT 的最终输入嵌入向量。

### 3.1 Token Embedding

Token Embedding 的核心作用是**将展平后的 Patch一维向量映射到指定的模型维度，并赋予初步的语义特征**，本质是一个线性投影操作，也被称为**PatchEmbedding**。

操作过程：展平后的 Patch 向量具有固定的维度（如 16×16×3 的 Patch展平后为 768 维），通过一个全连接层（线性层）将其映射到 ViT的模型隐藏维度 $d_{model}$（如 ViT-Base 的 $d_{model}=768$），若 Patch展平后的维度与模型隐藏维度一致，可直接跳过投影，仅做归一化处理；

核心意义：线性投影的过程并非简单的维度匹配，而是通过模型训练学习到 Patch的局部语义特征，将像素级的数值转化为具有视觉语义的向量表征，让每个 ImageToken 成为真正的 "视觉语义单元"。

Token Embedding 的输出为每个 Image Token 的语义嵌入向量，序列长度与Patch 数量一致，此时的 Token 仅具有语义信息，尚未包含空间位置信息。

### 3.2 Position Embedding

图像的 Patch 具有明确的空间位置关系（如左上角的 Patch、右下角的Patch），而 Transformer 的自注意力机制是**排列不变的**—— 若打乱Token的序列顺序，自注意力的计算结果不会发生变化，这会导致模型丢失图像的空间结构信息。因此，**PositionEmbedding**是 ViT 中不可或缺的部分，核心作用是**为每个 Image Token赋予唯一的空间位置信息**。

ViT 中的 Position Embedding 设计有两大核心特点：

1. **可学习的位置嵌入**：ViT 采用**可学习的一维位置嵌入**，而非 CNN的卷积偏置或 NLP中固定的正弦余弦位置嵌入。位置嵌入的维度与模型隐藏维度 $d_{model}$ 一致，其初始值为随机初始化，在模型训练过程中与其他参数一起迭代更新，让模型自主学习适合视觉任务的位置表征；
2.  **与 Token Embedding 相加融合**：将位置嵌入向量与对应的 Token Embedding向量做**逐元素相加**，而非拼接，最终得到同时包含**语义信息**和**空间位置信息**的嵌入向量，作为Transformer 编码器的输入。

此外，ViT 会在 Image Token 序列的最前端添加一个 **$[CLS]$ Token**，该Token无对应的Patch，仅为一个可学习的初始化向量，其位置嵌入也为单独的可学习向量，最终与其他Token 的嵌入向量一起输入编码器，用于后续的下游任务输出。

## 4. MLP分类头

上面通过TransformerEncoder后输出的shape和输入的shape是保持不变的，以ViT-B/16（ViT的base模型，Patch尺寸为16\*16）为例，输入的是 $[197,768]$ 输出的还是 $[197,768]$。这里我们只是需要分类的信息，所以我们只需要提取出 $[class]$ token生成的对应结果就行，即 $[197,768]$ 中抽取出 $[class]$ token对应的 $[1, 768]$。接着我们通过MLP Head得到我们最终的分类结果。MLP Head原论文中说在训练ImageNet21K时是由Linear+tanh激活函数+Linear组成。但是迁移到ImageNet1K上或者你自己的数据上时，只用一个Linear即可。

## 5. ViT 的完整工作流程

![workflow](assets/images/ViT/workflow.gif)
这里放一张作者提供的动图，非常清晰地展示了ViT的工作流程。

结合上述的 Patch Token 化、Embedding 层设计和 Transformer编码器架构，ViT的完整工作流程可分为**输入预处理**、**嵌入层融合**、**Transformer编码器特征提取**和**下游任务输出**四个阶段，以经典的图像分类任务为例，具体步骤如下：

### 阶段 1：输入图像标准化
将任意尺寸的输入 RGB 图像统一缩放到模型指定的固定尺寸（如224×224、384×384），并对像素值做归一化处理（如减均值、除标准差），确保输入数据的一致性。

### 阶段 2：Patch 切分与 Token 化
将标准化后的二维图像按固定大小切分为互不重叠的正方形 Patch，将每个 Patch展平为一维向量，形成 Image Token 序列。例如 224×224 的图像切分为 16×16的 Patch，得到 196 个 Image Token 的一维序列。

### 阶段 3：嵌入层融合

对 Image Token 序列做**Token Embedding**：通过线性层将展平后的 Patch向量映射到模型隐藏维度，得到语义嵌入向量；
添加 **$[CLS]$ Token**：在 Image Token 序列最前端拼接一个可学习的$[CLS]$Token 嵌入向量，序列长度变为 $196+1=197$；
叠加 **Position Embedding**：为每个 Token（包括\$[CLS]$ Token）分配唯一的可学习位置嵌入向量，与 Token 嵌入向量逐元素相加，得到最终的输入嵌入向量序列，维度为 $N \times d_{model}$（$N$为序列长度，$d_{model}$ 为模型隐藏维度）。

### 阶段 4：Transformer 编码器堆叠特征提取

将输入嵌入向量序列输入**堆叠的 Transformer编码器**中，每层编码器依次执行以下操作：

1. **多头自注意力机制**：通过多个注意力头并行计算每个 Token 与其他所有Token 之间的注意力权重，捕捉图像中 Patch之间的全局空间关联和语义依赖，实现特征的全局融合；
2. **残差连接与层归一化**：对自注意力的输出做残差连接（与输入嵌入向量相加），再进行层归一化，缓解模型训练中的梯度消失问题；
3. **前馈神经网络（FFN）**：由两层全连接层和激活函数（GELU）组成，对每个
Token 的特征做非线性变换，增强模型的特征表达能力；
4. **二次残差连接与层归一化**：对前馈神经网络的输出再次做残差连接和层归一化，得到该层编码器的输出特征。

经过多层编码器的处理后，输出的特征序列既包含每个 Patch 的局部语义特征，也融合了全局的空间关联特征。

### 阶段 5：图像分类任务输出

就用上面提到的MLP分类头。ViT 直接沿用 Bert 的 **$[CLS]$ Token**设计，仅提取$[CLS]$ Token对应的编码器输出特征，该特征已融合了整个图像的全局语义信息，无需再做全局平均池化等操作。将 $[CLS]$ Token 的输出特征输入一个**线性分类层**，通过 Softmax 函数转化为各类别的概率分布，概率最大的类别即为图像的分类结果。

若将 ViT应用于目标检测、语义分割等下游任务，仅需对输出层做相应改造：如目标检测任务可在Patch Token 的特征基础上添加检测头，语义分割任务则需将 Token 特征还原为二维的图像特征图。

## 6. ViT 的缺陷
毕竟这是一次比较大胆的尝试，把用于处理文字的Transformer拿来应用到图像领域，所以尽管ViT凭借全局自注意力机制在视觉任务中展现出优异的性能，但受限于架构设计和视觉任务的特性，ViT仍存在一些固有缺陷，在实际应用中需要针对性优化和改进：
- **低分辨率任务表现不佳**：ViT 的 Patch化处理会丢失图像的精细细节，当处理超分辨率重建、细粒度图像分类（如昆虫种类识别、文物细节检测）等低分辨率/细粒度视觉任务时，难以捕捉到关键的局部特征，表现不如专门针对细节提取优化的CNN 模型。
- **位置嵌入的泛化性差**：从我们上述的"从图片到token"的部分也了解到，ViT采用的可学习一维位置嵌入是针对固定尺寸图像训练的，当输入图像的尺寸与预训练尺寸不一致时，需要对位置嵌入进行插值缩放，或者对图片进行resize，这会导致位置信息的失真，大幅降低模型的泛化能力。而CNN 的卷积操作天然支持任意尺寸的图像输入，在尺寸适配性上更具优势。
- **对训练数据量要求极高**：ViT 摒弃了 CNN天然的局部归纳偏置（如平移不变性、局部相关性），无法像 CNN一样从少量数据中学习到视觉特征，必须依赖大规模的标注数据集进行预训练，若在小数据集上直接训练，性能会远不如CNN。这也导致 ViT的落地成本较高，对于数据量有限的小众视觉任务适配性较差。
- **计算与内存成本偏高**：尽管 Patch化已降低了自注意力的计算复杂度，但自注意力机制的平方级计算复杂度仍未根本改变。当ViT 处理大尺寸图像（如 512×512 及以上）时，Patch数量会大幅增加，序列长度变长，导致模型的计算量和内存占用急剧上升，对硬件设备的算力要求远高于同量级的CNN。

这就是ViT中存在的一些缺陷。经过了5年，已经有很多工作对它进行了改进，比如DeiT、Swin-Transformer、MAE
等众多改进模型，使它在视觉和多模态领域成为了不二之选。这些改进工作我也会写成博客，在之后的时间和大家见面，让你们了解到它的进化史，可能会对大模型有一个全新的认识。

## 7. 小结

ViT 作为视觉 Transformer 的开山之作，其设计思路极简而高效——**将图像序列化，用 Transformer 编码器做全局特征提取**，彻底摆脱了 CNN对局部卷积的依赖，通过自注意力机制实现了图像全局特征的直接建模。从Patch Token 化的图像预处理，到 Token Embedding 与 Position Embedding的特征融合，再到 Transformer 编码器的全局特征提取，每一步设计都围绕 "让Transformer 适配视觉任务" 展开。

ViT 的出现不仅推动了 CV 领域的技术革新，也构建了 NLP 与 CV领域的统一模型框架，为多模态大模型的发展奠定了基础。同时，我们也需认识到ViT存在对训练数据要求高、细粒度任务表现不佳、计算成本偏高等固有缺陷，而这些缺陷也成为后续视觉Transformer 改进的核心方向。
